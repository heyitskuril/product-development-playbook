## What is this?

This is a **free, open-source product development playbook** for developers, product managers, and indie hackers who want to build tech products the right way — with industry-standard practices at every phase.

It covers the complete lifecycle: from validating a problem to launching a production-ready product, with every phase backed by real references from industry-recognized books, standards, and organizations.

> No fluff. No opinion without citation. Just a practical, validated guide you can actually use.

## What's inside the Guide?

The [full guide](./GUIDE.md) covers **17 phases** of product development:

| # | Phase | Key Output |
|---|-------|-----------|
| 1 | 🧠 Product Discovery & Definition | Product Brief, Personas |
| 2 | 📋 Feature Scoping (MVP First) | Prioritized Feature List, User Stories |
| 3 | 🔄 User Flow & Journey Mapping | Flow Diagrams, Screen Inventory |
| 4 | 🧱 Data Modeling & Database Design | ERD, Data Dictionary |
| 5 | ⚙️ System Architecture | Architecture Diagram, ADRs |
| 6 | 🔐 Authentication & Authorization | Auth Flow, RBAC Matrix |
| 7 | 🧠 Business Logic Definition | Business Rules, State Machines |
| 8 | 🔌 API Design (Contract First) | OpenAPI Spec |
| 9 | 🎨 UI/UX Design | Wireframes, Design System |
| 10 | ⚠️ Edge Case & Failure Planning | Edge Case Checklist |
| 11 | 🗂️ Project Structure & Standards | Scaffold, CONTRIBUTING, README |
| 12 | 📅 Development Roadmap | Milestone Plan, Sprint Board |
| 13 | 🧪 Testing Strategy | Test Suite, Testing Plan |
| 14 | 🚀 Deployment Plan | CI/CD Pipeline, Runbook |
| 15 | 📊 Monitoring & Observability | Dashboards, Alerts |
| 16 | 🔒 Security Checklist | OWASP Coverage, Threat Model |
| 17 | 🔁 Post-Launch & Iteration | Post-Launch Report |

Each phase includes:
- ✅ **Actionable TODO checklist**
- 📌 **Concrete deliverable outputs**
- 📚 **Validated book references** (with year and rationale)
- 🔗 **Online resources** from recognized standards bodies (OWASP, W3C, NIST, Google, etc.)

## Quick Start

**Option A — Read it online**

Go straight to **[GUIDE.md](./GUIDE.md)** and start from Phase 1.

**Option B — Clone it locally**

```bash
git clone https://github.com/heyitskuril/product-development-playbook.git
cd product-development-playbook
```

Open `GUIDE.md` in any Markdown viewer, code editor, or Obsidian.

**Option C — Use it as a template**

Click **"Use this template"** (if enabled) or fork the repo and adapt it to your team's workflow.

## References & Standards

This guide is grounded in industry-recognized sources, including:

- [OWASP](https://owasp.org/) — Web application security standards
- [W3C WCAG 2.1](https://www.w3.org/WAI/WCAG21/) — Accessibility guidelines
- [NIST SP 800-63B](https://pages.nist.gov/800-63-3/sp800-63b.html) — Identity & password standards
- [OpenAPI 3.1 Specification](https://spec.openapis.org/oas/v3.1.0) — API contract standard
- [DORA Research](https://dora.dev/) — Engineering performance metrics
- [The 12-Factor App](https://12factor.net/) — Cloud-native application methodology
- [C4 Model](https://c4model.com/) — Architecture diagramming standard
- Books by Martin Fowler, Eric Evans, Martin Kleppmann, Marty Cagan, and more

Full references are listed at the end of [GUIDE.md](./GUIDE.md).
